## Purpose
_This directory contains:_ The **lib** module serves as the core library for the **auto-doc-tool** project, a Ruby-based automated documentation generator. Its purpose is to analyze a software project’s source code, extract structural and relational information, and produce comprehensive documentation outputs—including diagrams, index files, READMEs, and architecture overviews—often leveraging an LLM (Large Language Model) for summarization and natural language generation.

### Key Components

- **Orchestrator** (`orchestrator/`) – Manages the documentation generation workflow via a pipeline of configurable steps (e.g., `ArchitectureStep`, `DiagramStep`, `ReadmeStep`, `ManifestStep`, `IndexSummaryVectorsStep`, `MetricsHelper`).
- **Analyzer** (`analyzer/`) – Parses source code to extract imports, schemas, model associations, class hierarchies, and dependency information. Includes caching (`AnalysisCache`), diff analysis (`DiffService`), orphan detection (`OrphansService`), and YARD reading (`YardReader`).
- **Generator** (`generator/`) – Produces output documents and diagrams: README, architecture docs, C4 diagrams, class diagrams, ERDs, maps, indexes, summaries, vector embeddings, and more. Uses template helpers and Markdown formatting.
- **LLM Integration** (`llm/`) – Interfaces with a language model for summarization, prompt building, response parsing, and configuration management (`Client`, `PromptBuilder`, `ResponseParser`, `Summarizer`).
- **Transformer** (`transformer/`) – Converts raw analysis data into structured representations: class hierarchies, file trees, data flow graphs, ERD relationships, and graph data.
- **Utilities** (`utils/`) – Provides helpers for file tree building, YAML configuration loading, Markdown rendering, output formatting, and timestamp tracking.
- **CLI & Server** – `CLI.rb` offers a command-line interface, while `Server.rb` provides an HTTP service for interactive use.
- **Configuration & Indexing** – `Config.rb` manages project settings; `DocumentationIndex.rb` and `SearchService.rb` enable search over generated documentation.

The module is fully namespaced under `AutoDoc` and orchestrated by the main `Orchestrator.rb` file, which ties together the pipeline of analysis, transformation, and generation steps.
