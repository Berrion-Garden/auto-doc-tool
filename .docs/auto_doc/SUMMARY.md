3. **Analysis**: The `analysis_pipeline.rb` invokes specialized parsers (source, schema, YARD) to extract raw code data. Results are stored in `analysis_cache.rb`.

5. **LLM Calls**: When needed, steps call `agent_query_service.rb` which uses `llm.rb` → `client.rb`, `prompt_builder.rb`, and `response_parser.rb` to generate text descriptions or summaries.
6. **Generation**: Generators (e.g., `class_diagram_generator.rb`, `architecture_generator.rb`) take transformed data and produce output files (Markdown, Mermaid diagrams, etc.).
7. **Post-processing**: The `documentation_index.rb` is updated, and `reporter/` modules may produce audit reports.
8. **Output**: Generated files are written to the project directory (e.g., `docs/`), and optionally served via `server.rb`.

The design is modular and extensible: new analyzers, transformers, generators, or steps can be added by subclassing existing base classes.
