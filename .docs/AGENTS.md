## Project Overview

`auto-doc-tool` is an automated documentation generation utility designed to analyze codebases—primarily Ruby and Rails applications—and produce comprehensive, multi-format documentation suites. Its main purpose is to eliminate the manual overhead of maintaining project documentation by automatically extracting code structures, dependencies, and semantic context to generate up-to-date guides, architectural diagrams, and reference materials.

The project operates through a highly modular, step-by-step pipeline. The `Analyzer` module handles the initial codebase inspection, parsing source files, extracting model associations, reading database schemas, and processing existing inline comments. This structural data is then refined by the `Transformer` module, which builds internal representations like class hierarchies, dependency graphs, and data flows. Concurrently, the `LLM` module interfaces with large language models to enrich descriptions, summarize complex logic, and build context-aware prompts, ensuring the generated content is both accurate and highly readable.

The `Orchestrator` coordinates these analytical steps into a cohesive workflow, triggering specialized processing stages for different documentation needs. Finally, the `Generator` module exports the enriched data into multiple output formats, including Markdown files, C4 diagrams, class diagrams, Entity-Relationship Diagrams (ERDs), README overviews, and vector embeddings for semantic search. Supporting components like the `Reporter` module perform completeness audits, while the `CLI` and `Server` provide user-facing interfaces for running analyses and querying the generated documentation locally.

At its core, the application functions as a deterministic document factory: it ingests a target project, extracts structural and semantic metadata, builds relational graph representations, and finally synthesizes those graphs into documentation artifacts.
