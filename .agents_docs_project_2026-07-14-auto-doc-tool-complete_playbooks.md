# Auto-Doc Tool — Manual Test Playbooks

## Prerequisites
- [ ] Ruby gem installed or running from source (`ruby -I lib exe/auto-doc version`)
- [ ] Sample Ruby project exists at `fixtures/sample_ruby_project/`
- [ ] No pre-existing `.autodoc/` directory in test fixtures (clean baseline)

---

## Playbook FR-1: CLI End-to-End Generation
**Level:** 0 (infrastructure)
**Depends on:** nothing

### Play 1.1: Generate produces output tree
**Action:** Run `ruby -I lib exe/auto-doc generate fixtures/sample_ruby_project`
**Pass/Fail Criteria:**
- [ ] Command exits with code 0 (no crash)
- [ ] `.autodoc/` directory created inside fixtures/sample_ruby_project/
- [ ] `.autodoc/README.md` exists and contains project overview
- [ ] `.autodoc/diagrams/deps.mmd` exists and contains valid Mermaid syntax
- [ ] AGENTS.md created for each module root directory

### Play 1.2: Generate without config uses defaults
**Action:** Remove `.autodoc.yml`, run `generate` on a fresh project directory
**Pass/Fail Criteria:**
- [ ] Command completes without error
- [ ] Output uses default exclude patterns (spec/, test/, vendor/, node_modules/)

### Play 1.3: CLI flags override config
**Action:** Run `generate --exclude "docs" --output-dir .docs` on sample project
**Pass/Fail Criteria:**
- [ ] Output goes to `.docs/` directory instead of `.autodoc/`
- [ ] Specified exclude patterns are respected

### Play 1.4: Empty directory generates skeleton
**Action:** Run `generate` on an empty directory
**Pass/Fail Criteria:**
- [ ] Command completes without crash
- [ ] Output contains "No public symbols" placeholder text
- [ ] No Ruby file errors in console output

### Play 1.5: Self-generation produces self-documentation
**Action:** Run `generate` on auto-doc gem's own source
**Pass/Fail Criteria:**
- [ ] Produces `.autodoc/` with README.md for root
- [ ] AGENTS.md for lib/ directory
- [ ] No console errors

## Playbook FR-2: Hierarchical Directory Indexing
**Level:** 1 (core generation)
**Depends on:** FR-1

### Play 2.1: Index file at every directory level
**Action:** After generate, inspect `.autodoc/` tree
**Pass/Fail Criteria:**
- [ ] `_index.md` exists at the root `.autodoc/` level
- [ ] `_index.md` exists for each subdirectory (e.g., `.autodoc/lib/_index.md`)
- [ ] Root-level index lists all module roots with summaries

### Play 2.2: Index contains file tree and references
**Action:** Open root-level `_index.md`
**Pass/Fail Criteria:**
- [ ] Contains directory file tree in indented format
- [ ] Contains list of files with class/module names
- [ ] Contains public method counts per file
- [ ] Contains cross-reference links to subdirectory indexes

### Play 2.3: Single-file directory generates index
**Action:** Create a test directory with one file, run generate
**Pass/Fail Criteria:**
- [ ] `_index.md` created
- [ ] Index contains the single file listing
- [ ] No crash from single-file edge case

## Playbook FR-3: Semantic Search with Vector Embeddings

**Depends on:** FR-1

**Action:** Run `generate --embed` on sample project
**Pass/Fail Criteria:**
- [ ] `.autodoc/vector/` directory created

### Play 3.2: Search returns ranked results

**Pass/Fail Criteria:**
- [ ] Returns ranked list with at least one result
- [ ] Each result includes file path
- [ ] Each result includes relevance score

### Play 3.3: No-match query returns empty set
**Action:** Run `autodoc search "xyznonexistent12345"`
**Pass/Fail Criteria:**
- [ ] Returns empty result set (not an error)
- [ ] Message indicates no relevant documents found

- [ ] Second run is faster than first (fewer chunks processed)

## Playbook FR-6: Grep/Keyword Search Integration
**Level:** 2 (search features)
**Depends on:** FR-1, FR-3

### Play 6.1: Keyword search returns exact matches
**Action:** Run `autodoc search --keyword "User"` on sample project
**Pass/Fail Criteria:**
- [ ] Returns files containing "User" as a symbol name
- [ ] Each result shows file path and line number
- [ ] Each result shows matching context line

### Play 6.2: Case-insensitive matching
**Action:** Run `autodoc search --keyword "user"`
**Pass/Fail Criteria:**
- [ ] Returns same results as uppercase "User"
- [ ] "user", "User", "USER" all match

### Play 6.3: Partial keyword matching
**Action:** Run `autodoc search --keyword "auth"`
**Pass/Fail Criteria:**
- [ ] Returns results containing "authenticate", "authorization", "auth" variants
- [ ] Partial matches ranked below exact matches

### Play 6.4: Combined semantic + keyword search
**Action:** Run `autodoc search --keyword "User" --hybrid` (or default hybrid mode)
**Pass/Fail Criteria:**
- [ ] Results from both semantic and keyword search merged
- [ ] Final ranking uses Reciprocal Rank Fusion
- [ ] No duplicate results

### Play 6.5: Empty keyword search
**Action:** Run `autodoc search --keyword "zzz_not_a_real_symbol_9999"`
**Pass/Fail Criteria:**
- [ ] Returns empty result set
- [ ] Clear message, not an error

## Playbook FR-7: Agent Query Interface
**Level:** 2 (agent API)
**Depends on:** FR-1, FR-2, FR-3, FR-4

### Play 7.1: Find symbol by name
**Action:** Run `autodoc query --find-symbol "User" --format json`
**Pass/Fail Criteria:**
- [ ] Returns valid JSON
- [ ] Contains `file_path`, `line_number`, `type` (class/module/method)
- [ ] Contains `doc_string` if available
- [ ] Contains `scope_chain` (parent modules/classes)

### Play 7.2: Find dependents of a module
**Action:** Run `autodoc query --find-dependents "app/models/user.rb" --format json`
**Pass/Fail Criteria:**
- [ ] Returns list of files that import or reference the module
- [ ] Each entry includes file path and import type

### Play 7.3: Find dependencies of a module
**Action:** Run `autodoc query --find-dependencies "app/models/user.rb" --format json`
**Pass/Fail Criteria:**
- [ ] Returns list of files the module imports
- [ ] Each entry includes file path and import type

### Play 7.4: Coverage query returns structured stats
**Action:** Run `autodoc query --coverage --format json`
**Pass/Fail Criteria:**
- [ ] Returns JSON with per-directory coverage data
- [ ] Includes aggregate coverage percentage
- [ ] Includes total symbol counts

### Play 7.5: Summary query returns structured module data
**Action:** Run `autodoc query --summary --dir "lib" --format json`
**Pass/Fail Criteria:**
- [ ] Returns JSON with classes, methods, dependencies
- [ ] Includes purpose text
- [ ] Includes doc coverage percentage

### Play 7.6: Invalid query type returns error
**Action:** Run `autodoc query --find-symbol --invalid-flag --format json`
**Pass/Fail Criteria:**
- [ ] Returns error JSON with `error` key
- [ ] Error message lists valid query types
- [ ] Exit code 1

### Play 7.7: Symbol not found returns suggestions
**Action:** Run `autodoc query --find-symbol "NonExistentClass999" --format json`
**Pass/Fail Criteria:**
- [ ] Returns `{"found": false}`
- [ ] Includes `suggestions` array with close matches (if any)

### Play 7.8: HTTP endpoint serves query results
**Action:** Start `autodoc serve`, then `curl http://localhost:4567/api/query?type=find-symbol&name=User`
**Pass/Fail Criteria:**
- [ ] Returns JSON equivalent to CLI query result
- [ ] HTTP status 200
- [ ] Response includes same data as CLI version

## Playbook FR-8: Vector/Summary/Index Mapping
**Level:** 2 (cross-reference data)
**Depends on:** FR-1, FR-3, FR-4

### Play 8.1: Map file generated after full generate
**Action:** Run `generate --embed`, check for map file
**Pass/Fail Criteria:**
- [ ] `.autodoc/map.json` exists
- [ ] File is valid JSON (parseable)

### Play 8.2: Map contains all required sections
**Action:** Read and parse `.autodoc/map.json`
**Pass/Fail Criteria:**
- [ ] Contains `schema_version` field (value "1.0")
- [ ] Contains `vector_chunks` array with chunk entries
- [ ] Each chunk entry has: `chunk_id`, `source_file`, `doc_file`, `line_range`, `symbol_name`
- [ ] Contains `directory_indexes` listing all `_index.md` paths
- [ ] Contains `diagrams` listing all `.mmd` file paths
- [ ] Contains `module_roots` listing all module root paths

### Play 8.3: Map without embeddings still valid
**Action:** Run `generate` (without --embed), inspect map
**Pass/Fail Criteria:**
- [ ] `.autodoc/map.json` exists
- [ ] `vector_chunks` field is empty array (not missing)
- [ ] `directory_indexes` and `diagrams` still populated
- [ ] File is valid JSON

### Play 8.4: Corrupt map regenerates on next run
**Action:** Corrupt `map.json` with invalid content, run `generate` again
**Pass/Fail Criteria:**
- [ ] Console warns about previous corruption
- [ ] New valid `map.json` replaces corrupt one
- [ ] Generate completes without crash

## Playbook FR-9: Documentation Coverage Audit
**Level:** 0 (infrastructure)
**Depends on:** nothing

### Play 9.1: Audit produces text table
**Action:** Run `autodoc audit fixtures/sample_ruby_project`
**Pass/Fail Criteria:**
- [ ] Output contains "Auto-Doc Audit Report" header
- [ ] Output contains coverage percentage
- [ ] Output contains "PASS" or "FAIL" based on threshold

### Play 9.2: Audit produces JSON report
**Action:** After audit, inspect `report.json`
**Pass/Fail Criteria:**
- [ ] `report.json` exists in project directory
- [ ] Contains `total_symbols`, `documented`, `undocumented`, `coverage_percent`
- [ ] Contains `passed_threshold` boolean
- [ ] Contains `failures` array (may be empty)
- [ ] JSON is valid and parseable

### Play 9.3: Threshold gating works
**Action:** Run `audit --threshold 100` on partially documented project
**Pass/Fail Criteria:**
- [ ] Exit code 1 (failure)
- [ ] Output shows "FAIL"
- [ ] Failures list undocumented symbols

### Play 9.4: Threshold pass works
**Action:** Run `audit --threshold 0` on same project
**Pass/Fail Criteria:**
- [ ] Exit code 0 (pass)
- [ ] Output shows "PASS"

### Play 9.5: Empty project always passes
**Action:** Run `audit --threshold 100` on empty directory
**Pass/Fail Criteria:**
- [ ] Exit code 0
- [ ] Coverage shows 100% (0 of 0 symbols)

## Playbook FR-10: Orphan File Detection
**Level:** 1 (dead code analysis)
**Depends on:** FR-1

### Play 10.1: Orphans command lists disconnected files
**Action:** Run `autodoc orphans fixtures/sample_ruby_project`
**Pass/Fail Criteria:**
- [ ] Lists files with zero import edges (inbound and outbound)
- [ ] Each result shows file path and symbol count
- [ ] Test/spec directories excluded from results

### Play 10.2: Entry point files annotated
**Action:** Create a fixture with a config.ru or Rakefile as entry point, run orphans
**Pass/Fail Criteria:**
- [ ] Entry point listed with "(entry point)" annotation
- [ ] Still classified as orphan but clearly marked

### Play 10.3: No orphans returns clean
**Action:** Create fixture where every file is imported, run orphans
**Pass/Fail Criteria:**
- [ ] Exit code 0
- [ ] Message: "No orphan files found"

### Play 10.4: All orphans in new project
**Action:** Create fresh directory with .rb files, no cross-imports, run orphans
**Pass/Fail Criteria:**
- [ ] Exit code 2 (orphans found)
- [ ] All files listed
